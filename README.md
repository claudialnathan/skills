SKILLS █

─────────────────────

Applied skills built from [claudialnathan/agent-kitchen](https://github.com/claudialnathan/agent-kitchen) that have survived several seasons of models and audits.

These are skills that add meaningful value to my work in a way that is better than the unaided model can do out of the box.

The [kitchen](https://github.com/claudialnathan/agent-kitchen) is built on the idea that a frontier model in 2026 is very competent at most things, but its default in any domain is the competent-average version of the thing.

Each skill here is a commitment that drags the model off that median in a chosen direction. Each competes with two forces:

- the unaided frontier model, which absorbs generic craft with each release
- my time, which because every artifact charges rent on, deletion is the expected end of every skill here, not a failure of one.

Each skill earns its place only by beating the unaided current model on the gap that created it, checked by a runnable probe (`evals/probes.md`).

On each model release the probes replay against the bare model via the repo's `/retest` procedure (`.claude/skills/retest`); verdicts land in each probe file's baseline table and `CHANGELOG.md`. When every probe passes unaided across repeat runs, the model has absorbed the skill's job and the skill is deleted. Process skills (changelog, quality-audit) are exempt — their value is the owner wanting the procedure, which no release absorbs.

#### The Skills

- SHADCN-TAILWIND — stack-wide UI discipline for shadcn 4 (Base UI) + Tailwind v4:
  component architecture and token mechanics.
- DESIGN-LAYOUT — layout primitives, fluid type/space at the token layer, container
  queries, mobile viewport units. (Carved from design-engineer, 2026-07-15.)
- DESIGN-MOTION — frequency-aware motion: the frequency × novelty master rule, the
  framework-native hierarchy of reach, Motion + Base UI integration.
- DESIGN-POLISH — the proactive detail layer applied unprompted; owns the pre-ship
  UI checklist.
- DESIGN-TASTE — the judgment layer: state the reason, precise vocabulary, anti-slop.
- SPEED-DAEMON — optimistic-UI patterns where perceived speed matters (the component
  layer, not a sync engine).
- ADVISOR — survey a codebase as a senior advisor; write self-contained plans a
  cheaper executor model runs. (Forked from shadcn's improve, 2026-07-15.)
- QUALITY-AUDIT — stack-aware, read-only quality audit for a JS/TS web repo.
- SALTINTESTA — prose tone: articulate ideas in as few good words as possible.
- FLAVORED-MD — GitHub-Flavored Markdown, with the form matched to the content's job.
- ZOOM-OUT — a zoomed-out strategic review of a whole project against its actual purpose.
- CHANGELOG — a skimmable, append-only decision log of what changed and why.
