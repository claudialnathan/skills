## 2026-07-14 — worklog retired; `changelog` folded in from machine scope as the lightweight decision log

- **worklog + worklog-minor deleted; replaced by a single `changelog` skill.** The worklog system (`.worklog/BRIEF.md` derived from an append-only `LOG.md`, four entry types, a significance gate, and inline graduation to `AGENTS.md`/`.claude/rules`) was more machinery than the everyday gap needs. Owner asked for something lightweight, low-token, scannable by human and agent, that does not overlap with git history, auto memory, or transcripts. The heavy pieces (BRIEF projection, graduation) were the cost; the value was the dated why/rejected/open record. `changelog` keeps the value and drops the machinery.
- **The `/changelog` this repo already referenced was a phantom in-repo but real at machine scope.** worklog's SKILL.md deferred "what changed" to "the `/changelog` skill" — no such skill existed in `serve/`. It existed as a standalone real dir at `~/.claude/skills/changelog/` (authored 2026-05-27, single SKILL.md), which is the skill a stackradar session correctly reported as "installed." Personal scope overrides plugin scope, so shipping `serve/skills/changelog` while that copy lives would let the older machine-scope copy silently shadow the kitchen one everywhere. Folded the machine-scope skill into `serve/` as canonical; owner removes the machine-scope copy (below) so the plugin version wins.
- **Scope was set by investigation, not by the request.** Checked the mid-2026 state before building: a live movement (Contextual Commits open standard 2026-03-14, "Lore", Git Context Controller) argues the why/rejected/learned belong in enriched git commit bodies, not a separate file; ADRs remain the committed-markdown standard for why + rejected but are critiqued as going stale against the live system; AGENTS.md cannot absorb history (instruction-following degrades past ~120 lines of project context). Contextual Commits' own month-1 field data undercuts the pure-git position: 72.5% of commits adopted the `[context]` scaffold but only 11.8% filled `rejected:` — a 2.6× structure-vs-substance gap ("hollow compliance"), self-described as "a signal, not proof." Conclusion: a committed file earns its place, but only for the three things git (even enriched) structurally cannot give — rejected approaches that left no commit, the still-open dimension that has no commit to attach to, and a curated newest-first skim surface. The skill pulls Files from `git diff` and writes only Why/Rejected/Open from the session, so it complements git rather than duplicating it.
- **Improvements over the standalone on fold-in.** (1) Lightweight Rejected-pushback: before logging, skim the sparse `**Rejected:**` lines and surface the dated entry if the new work repeats a rejected approach — preserves worklog's most valuable behavior (stop re-walking a dead end) at near-zero read cost, and directly answers the "agents repeat rejected approaches" problem the research names. (2) Broadened collision guard: the standalone only stopped on release-changelog markers; it now also stops on any other structured log (provenance/model-pin ledger, auto-generated file) — which protects this very repo's `CHANGELOG.md` (a provenance ledger) from being appended to if `/changelog` ever runs here. (3) Explicit complements-git-not-a-second-git framing up top. (4) Kitchen provenance pin + README; worklog comparisons removed from body and description.
- **Process skill — probe-exempt.** Like the worklog it replaces, its value is the owner wanting the procedure, which no model release absorbs. CLAUDE.md's process-skill exemption list updated (`worklog, worklog-minor` → `changelog`); `serve` plugin.json and marketplace.json descriptions updated.
- **Machine scope reported, not touched (forge-family boundary).** Owner to run: `rm -rf ~/.claude/skills/changelog` (standalone, now superseded) and `rm ~/.claude/skills/worklog ~/.claude/skills/worklog-minor` (symlinks into `serve/`, now dangling). The kitchen `changelog` reaches other repos through the plugin (push → `/plugin marketplace update claudia` → `/plugin update`), not a machine-scope symlink; `bin/sync-cross-tool` handles `~/.cursor`, `~/.codex`, and repo-local `.claude/skills` and prunes the worklog orphans there.
